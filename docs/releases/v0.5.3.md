# SessionGuard 0.5.3

Operator UX refinement patch release focused on making approval timing, service mode, and policy transitions easier to understand without keeping the full dashboard open all the time.

## Highlights

- added first-class approval-timing text in the dashboard and tray surface
- added tray balloon notifications for service fallback, service reconnection, approval timing events, and policy transitions
- added empty-state messaging so quiet dashboard sections do not look broken
- fixed approval messaging so cleared approval windows are not mislabeled as expired

## Included in this release

- new operator alert models and evaluator in `SessionGuard.Core`
- tray menu status, mode, policy, and timing lines in `SessionGuard.App`
- approval-expiry warning lead time in `config/appsettings.json`
- new tests covering approval expiry, early approval clearing, fallback notifications, and policy configuration transition alerts
- updated UI smoke expectations for policy timing text
