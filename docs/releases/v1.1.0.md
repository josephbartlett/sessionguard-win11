# SessionGuard 1.1.0

Tray-first experience release for the `1.x` line.

## Highlights

- the desktop app now behaves more like a calm tray utility and less like a dashboard-first console
- repeated launches reuse the existing tray app instead of opening duplicate instances
- installed startup behavior is quieter and better aligned with the intended app-plus-service model
- the main window now uses a tighter header/action layout with clearer visual hierarchy

## Included in this release

- single-instance app coordination in `SessionGuard.App`
- tray summary, next-step, and context text driven from the shared operator alert model
- contextual primary tray action support
- updated startup and install validation for the tray-first path
- refreshed runtime and operator docs that explain the tray-first behavior explicitly
- additional tests and UI smoke coverage for tray-first startup and overview rendering

## Architecture

- the service remains the authoritative runtime for machine-level monitoring and state-changing actions
- the app is now more explicitly the tray and dashboard client layered over that runtime
- the tray-first shift changes presentation and startup behavior without weakening the existing service boundary

## Limitations

- service-backed writes still require the service path and appropriate app elevation
- the tray shell is quieter, but deeper diagnosis still depends on opening the main window
- SessionGuard remains a direct-download Windows utility rather than a Microsoft Store package
