# SessionGuard 1.0.1

Documentation and release-quality cleanup for the `1.0.x` line.

## Highlights

- shorter, clearer README focused on first-run success
- new getting-started and manual-validation guides
- tighter documentation tone for a post-`1.0.0` product
- cleaned-up repo guidance for documentation, links, and current-state accuracy

## Included in this release

- rewritten top-level `README.md`
- new `docs/getting-started.md`
- new `docs/manual-validation.md`
- refreshed architecture, roadmap, product brief, limitations, and future-service notes
- one user-visible mitigation description cleanup so the running product no longer refers to itself as an MVP instance

## Architecture

- no architectural change to the service, control-plane, policy, or workspace model
- the release improves how the product is explained and operated rather than changing core behavior

## Limitations

- this release does not expand product scope
- SessionGuard remains a direct-download Windows utility rather than a Microsoft Store-focused build
- restart mitigation boundaries and platform limits are unchanged from `1.0.0`

## Verification

- `dotnet build SessionGuard.sln`
- `dotnet test SessionGuard.sln`
- GitHub Actions validation passed for the documentation and release-metadata commits on this branch
