# SessionGuard 0.3.1

Patch release focused on finishing the `0.3.x` stabilization work after the initial background-protection architecture landed in `v0.3.0`.

## Highlights

- Added persistent service health snapshots for startup, scan, and pipe diagnostics
- Added install-readiness validation without modifying the machine
- Added repo-external published-layout validation for the service package
- Expanded automated coverage for PowerShell service scripts and runtime path behavior
- Made local fallback wording in the desktop UI more explicit

## Included in this release

- `state/service-health.json` written by the service host
- `SessionGuard.Service.exe health` command for reading the persisted health snapshot
- `Install-SessionGuardService.ps1 -ValidateOnly` for non-destructive readiness checks
- `Validate-SessionGuardPublishedLayout.ps1` for repo-external publish validation
- additional tests for:
  - service health reporting
  - runtime path resolution
  - service script JSON and validation behavior

## Architecture

- `SessionGuard.Service` now publishes a machine-readable health snapshot alongside the scan snapshot
- service tooling consumes health data directly instead of inferring service state only from logs
- the stabilization track now covers published-layout validation and install-readiness checks in addition to the earlier IPC and service split work

## Limitations

- clean-machine service installation still requires an elevated manual validation pass on a real Windows 11 environment
- the desktop client still uses a fallback in-process scan path when the service is unavailable
- this release does not start the `v0.4.0` workspace-safety feature work yet

## Verification

- `powershell -ExecutionPolicy Bypass -File scripts/service/Validate-SessionGuardPublishedLayout.ps1`
