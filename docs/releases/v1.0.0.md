# SessionGuard 1.0.0

Production baseline release for SessionGuard on Windows 11.

## Highlights

- service-backed restart awareness with a tray-aware desktop client
- restart-signal aggregation across registry, Windows Update Agent, UX settings, and scheduled-task visibility
- workspace-risk heuristics for terminals, browsers, editors, and local-runtime style activity
- JSON-backed policy engine with blocking rules, restart windows, and temporary approval windows
- calmer overview-first dashboard with explicit `Simple view` and `Technical view` modes
- upgrade-safe service deployment with runtime validation, config migration, and one-command update orchestration
- deterministic local and CI validation, including WPF UI smoke automation
- automated binary release assets for desktop app, service, and source packages on version tags

## Included in this release

- `SessionGuard.App` WPF dashboard and tray-aware shell
- `SessionGuard.Service` background worker and named-pipe control plane
- `config/protected-processes.json`, `config/appsettings.json`, and `config/policies.json`
- local service-management scripts for publish, install, update, status, validation, and uninstall
- local desktop-app publishing and release-asset packaging scripts
- local structured logs, scan snapshots, workspace snapshots, policy approval persistence, and service-health snapshots

## Operational boundaries

- SessionGuard does not disable Windows Update.
- SessionGuard does not guarantee prevention of every restart path Windows can take.
- Workspace protection is advisory; SessionGuard does not restore unsaved application state.
- Mitigation and approval writes require the background service path and appropriate elevation.
- The full release is best suited to direct download distribution; the current service-plus-elevation model is not a clean Microsoft Store fit without a reduced packaging approach.
