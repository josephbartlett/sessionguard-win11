# SessionGuard 0.5.2

Service-boundary tightening patch release focused on making the background service the authoritative owner for mitigation writes and temporary approval changes.

## Highlights

- made mitigation apply/reset and restart-approval grant/clear actions service-owned
- kept local fallback available for monitoring while making those write actions explicitly read-only
- tightened hybrid control-plane fallback so only true service unavailability triggers local fallback
- added service startup approval-state recovery details to `state/service-health.json`

## Included in this release

- new control-plane unavailable exception for transport and protocol availability failures
- disabled local write execution for mitigation and approval actions
- updated WPF messaging so local fallback clearly explains why write actions are unavailable
- protocol version bump to `1.2`
- expanded tests for service-only actions, remote application failures, approval recovery, and IPC payload updates
