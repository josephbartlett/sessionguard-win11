# SessionGuard 0.5.1

Policy hardening patch release focused on making the new rule engine safer to operate when policy configuration is malformed, conflicting, or ambiguous.

## Highlights

- added policy configuration validation and dashboard diagnostics
- kept restart scanning and workspace monitoring alive when `config/policies.json` is malformed
- made approval-window precedence deterministic and explicit when multiple approval rules match
- added clearer policy evaluation trace messaging for blocking-versus-approval overlap

## Included in this release

- new policy validation report and issue models in `SessionGuard.Core`
- new policy configuration validator for duplicate IDs, disabled rules, and conflicting window warnings
- safe fallback behavior in `JsonConfigurationRepository` for malformed policy JSON
- updated policy card UI with config-health summary, diagnostics, and evaluation trace output
- expanded unit tests for policy validation, approval-window precedence, expired approval cleanup, and pipe payload round-trips
