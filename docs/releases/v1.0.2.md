# SessionGuard 1.0.2

Combined install and distribution cleanup for the `1.0.x` line.

## Highlights

- single combined install path for the app and service
- root-level bundle install and uninstall entry points in the downloaded release bundle
- explicit runtime-model documentation for app, service, tray, and startup behavior
- bundled release asset that packages both runtimes together for real-machine installs

## Included in this release

- new `scripts/install/Install-SessionGuard.ps1`
- new `scripts/install/Uninstall-SessionGuard.ps1`
- new `scripts/install/Publish-SessionGuardBundle.ps1`
- startup registration for `SessionGuard.App.exe` at user sign-in during combined install
- release asset packaging that now emits a combined bundle zip in addition to separate app, service, and source assets
- new `docs/runtime-model.md` and updated setup documentation across README and getting-started flows

## Architecture

- the service remains the background engine and machine-level auto-start component
- the app remains the tray and dashboard shell and is now registered to start at sign-in through the combined installer
- release packaging now has an operator-friendly combined bundle without changing the underlying two-process runtime model

## Limitations

- the combined install registers app startup for the current user only
- SessionGuard still uses separate app and service processes by design; the combined bundle is packaging convenience, not a merged runtime
- service install and state-changing operations still require elevation
