# SessionGuard 1.1.2

Patch release focused on making the public repository easier to understand by simplifying the README and moving contributor-facing detail into deeper docs.

## Highlights

- simplified the top-level README into a clearer front-door document
- added a dedicated development and release guide for source-build and packaging workflows
- tightened the getting-started guide so it stays focused on install and common operator paths

## Included in this release

- updated `README.md` to focus on product summary, preferred install, runtime at a glance, limits, and doc links
- added `docs/development.md`
- trimmed `docs/getting-started.md` to remove release-engineering and packaging detail that now lives in the development guide

## Architecture

- no runtime architecture changes in this release
- the app, service, control-plane, packaging, and validation behavior are unchanged
- this release improves how the existing product is explained and navigated

## Limitations

- this release does not expand SessionGuard's technical scope
- service-backed writes, tray startup, and packaging constraints remain the same as `1.1.1`
- SessionGuard remains a direct-download Windows utility rather than a Microsoft Store package

## Verification

- `dotnet build SessionGuard.sln`
- `dotnet test SessionGuard.sln`
- `powershell -ExecutionPolicy Bypass -File scripts/ci/Invoke-WindowsValidation.ps1`
- `powershell -ExecutionPolicy Bypass -File scripts/release/Publish-SessionGuardReleaseAssets.ps1 -Version 1.1.2 -Configuration Release -Runtime win-x64 -SelfContained`
