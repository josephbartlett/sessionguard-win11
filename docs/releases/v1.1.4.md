# SessionGuard 1.1.4

Patch release focused on hardening the combined setup flow for real reinstall and upgrade use on Windows 11.

## Highlights

- stops the running installed tray app before replacing files during reinstall or upgrade
- keeps combined uninstall aligned with the same installed-app shutdown logic
- treats blocked first-launch prompts as a warning instead of an ambiguous failure
- refreshes persisted service health metadata correctly after upgrade instead of preserving the previous install version
- documents the current unsigned first-launch SmartScreen expectation more explicitly

## Included

- shared installed-app stop helper in `scripts/install/common.ps1`
- combined installer update path in `scripts/install/Install-SessionGuard.ps1`
- combined uninstall reuse of the same shutdown helper in `scripts/install/Uninstall-SessionGuard.ps1`
- updated install guidance in `README.md`, `docs/getting-started.md`, and the generated bundle README
- regression coverage in `tests/SessionGuard.Tests/ServiceScriptTests.cs`

## Verification

- `dotnet build SessionGuard.sln`
- `dotnet test SessionGuard.sln`
- `powershell -ExecutionPolicy Bypass -File scripts/ci/Invoke-WindowsValidation.ps1`
- fresh bundle publish plus a real install-over-existing validation cycle
