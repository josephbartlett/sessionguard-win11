# SessionGuard 0.3.0

Background Protection Architecture release focused on turning the earlier service foundation into an operator-usable split between a continuous service host and a tray-aware desktop client.

## Highlights

- Added a versioned named-pipe control plane for status and mitigation requests
- Added a tray-aware WPF dashboard that prefers the service and falls back locally when necessary
- Added operator scripts for publishing, installing, starting, stopping, querying, and uninstalling the Windows Service
- Added separate app and service log streams for clearer diagnostics
- Added release packaging and source-version metadata for the 0.3.0 milestone

## Included in this release

- `SessionGuard.Service` control-plane commands:
  - `probe`
  - `status`
  - `scan-now`
- Service operations scripts under `scripts/service/`
- Release packaging script at `scripts/package-release.ps1`
- Shared protocol version validation in the pipe transport
- Expanded test coverage for control-plane fallback and IPC compatibility failures

## Architecture

- `SessionGuard.Service` now owns the authoritative background scan loop and named-pipe server
- `SessionGuard.App` acts as a tray-aware dashboard client over a hybrid control plane
- `SessionGuard.Infrastructure` now carries the versioned IPC envelope and separate app/service logging
- Release metadata is centralized in `Directory.Build.props`

## Limitations

- Service installation still requires elevation and is intended for local power-user validation rather than enterprise rollout
- The WPF shell is tray-aware, but it is not yet a dedicated lightweight tray-only client
- IPC is now versioned, but the protocol is still local-only and not designed for remote management
- Workspace protection remains process-based and does not capture unsaved application state
