# SessionGuard 0.5.0

Policy Engine release focused on turning SessionGuard from a fixed restart-awareness dashboard into a rule-driven local restart-governance tool.

## Highlights

- added a separate `config/policies.json` schema for restart windows, process or workspace restart blocks, and approval-required rules
- added deterministic policy evaluation with matched-rule explanations and approval-window state
- added persisted temporary restart approval windows shared across the app and service paths
- updated the WPF dashboard and control plane to expose policy state and approval actions directly

## Included in this release

- new policy configuration and evaluation models in `SessionGuard.Core`
- new local policy approval persistence at `state/policy-approval.json`
- new control-plane methods to grant and clear temporary restart approval
- new service utility commands:
  - `approve-restart`
  - `clear-approval`
- updated UI smoke scenarios and tests to cover policy decision states

## Architecture

- policy configuration now lives separately from `appsettings.json` and protected-process configuration
- the coordinator now evaluates policy rules against restart state, workspace state, observed processes, and approval-window state
- the app and service share the same policy evaluation output and approval persistence
- the WPF dashboard now renders policy decision, approval state, and matched rules as first-class status surfaces

## Limitations

- policy rules are enforced by SessionGuard logic, not by Windows itself
- temporary approval windows are local-only and single-machine; there is no shared approval authority yet
- the current policy engine is intentionally bounded to local JSON configuration and a small rule vocabulary
