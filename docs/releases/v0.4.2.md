# SessionGuard 0.4.2

Validation Hardening release focused on making the repo's Windows validation path reproducible in both local development and GitHub Actions.

## Highlights

- added a repo-owned Windows validation script that runs build, test, and UI smoke through one command
- added a GitHub Actions workflow on `windows-latest` to run that validation path and publish artifacts
- updated the UI smoke script so CI can reuse existing build output instead of rebuilding unnecessarily
- tightened repo guidance so UI-affecting changes require the same validation path and screenshot review

## Included in this release

- new `scripts/ci/Invoke-WindowsValidation.ps1`
- new `.github/workflows/windows-validation.yml`
- updated `scripts/ui/Run-UiSmoke.ps1` for configuration selection and `-SkipBuild`
- updated `AGENTS.md`, `README.md`, and `docs/roadmap.md` to point at the shared validation entry point
- ignored `artifacts/ci/` so local validation leaves the worktree clean

## Architecture

- validation logic now lives in repo-owned PowerShell scripts rather than in GitHub Actions YAML
- GitHub Actions is now a thin wrapper that calls the same validation path used locally
- UI smoke artifacts, test results, and summary metadata are grouped under `artifacts/ci/windows-validation/`

## Limitations

- the first true GitHub Actions proof still depends on a pushed run
- the workflow validates launch, controls, screenshots, and summary artifacts, but it is not a full end-to-end product workflow suite
- UI screenshot review is still heuristic and not yet a pixel-diff system
